# Comprehensive E2E Testing Plan

## Overview
Add thorough testing at two layers: **API integration tests** (Vitest + fastify.inject) for all 50+ endpoints across 12 modules, and **browser e2e tests** (Playwright) for the full operator workflow. Tests use inline helper functions and a dedicated test database.

## Interview Summary
- **Layers**: Both API integration (fastify.inject) + browser e2e (Playwright)
- **Coverage**: Everything deep — all endpoints with success, error, and edge cases
- **Test data**: Inline helpers per test (extend existing pattern)
- **API style**: fastify.inject() — tests real routing, middleware, auth, validation without starting a server
- **Fastify instance**: One per test suite, DB cleared between tests
- **Playwright orchestration**: webServer config auto-starts API + web
- **UI flows**: Full operator workflow (auth, CRUD, transactions, invoices, settlements, reconciliation, period)
- **CI**: Not now, local infrastructure only

## Current State
- Vitest installed + configured in apps/api
- 3 existing test files (distribute, ledger, invoice) calling services directly
- Test DB via Docker Compose (postgres-test on port 5435)
- test-setup.ts with migrations + table cleanup
- Inline helpers pattern (createTestUser, createTestEntity, etc.)
- **No route-level tests** — current tests bypass HTTP layer
- **No web tests** — zero test infrastructure for Next.js app
- **No Playwright** installed

## Desired End State
- API: Every endpoint has at least success + validation error + auth error test cases
- API: Critical flows (transactions, invoices, settlements) have deep edge case coverage
- Browser: Playwright tests cover full operator workflow end-to-end
- Shared test helpers in a dedicated file (not duplicated across tests)
- All tests run via `pnpm test` (API) and `pnpm test:e2e` (browser)

## What We're NOT Doing
- CI/CD pipeline (deferred)
- Visual regression testing
- Performance/load testing
- Mobile viewport testing
- Viewer role browser tests (API tests cover permissions)

**File**: `apps/api/src/build-app.ts` (NEW)
**Changes**: Factory that creates a fully-configured Fastify instance for testing:

```ts
// Builds the Fastify app with all plugins and routes registered.
// Used by test suites with fastify.inject() — no server.listen() needed.
import Fastify from 'fastify';
import cookie from '@fastify/cookie';
import cors from '@fastify/cors';
// ... import all route modules (same as index.ts)

export async function buildApp() {
  const server = Fastify({ logger: false });
  
  // Register plugins (same as index.ts but no helmet, no rate-limit, logger: false)
  await server.register(cookie);
  await server.register(cors, { origin: true, credentials: true });
  
  // BigInt serialization hook (same as index.ts)
  server.addHook('preSerialization', async (_request, _reply, payload) => {
    return JSON.parse(JSON.stringify(payload, (_key, value) =>
      typeof value === 'bigint' ? Number(value) : value
    ));
  });

  // Register all routes (same prefixes as index.ts)
  await server.register(authRoutes, { prefix: '/api/auth' });
  await server.register(entityRoutes, { prefix: '/api/entities' });
  // ... all other routes
  
  // Error handler
  server.setErrorHandler(errorHandler);

  await server.ready();
  return server;
}
```

## Edge Cases Addressed
- Empty accountId in transaction entries (the bug that triggered this)
- Token rotation on refresh (reuse detection)
- Double-entry balance invariant
- Closed period rejection
- Duplicate invoice collection
- Ownership percentage validation (must sum to 10000)
- Settlement rounding (largest remainder method)
- Reconciliation with discrepancy detection

## Testing Strategy
- **API layer**: Vitest + fastify.inject(), test DB cleared between tests
- **Browser layer**: Playwright, servers auto-started, seed data via API calls
- **Run locally**: `pnpm test` (API) + `pnpm --filter @financiero/web test:e2e` (browser)
- **Isolation**: Each test creates its own data, no shared state between tests
